1️. File Path System

A file path tells the browser where to find your files (HTML, CSS, images, etc.). Without correct paths, files won’t load. There are two main types:

Relative Paths (for your own project files)

Relative paths are based on the location of your HTML file.

  • Same folder: If about.html is in the same folder as your main file: <a href="about.html">About</a> This opens about.html in the same folder.
  • Sub-folder: If your image is inside an images folder: <img src="images/logo.png" alt="Logo"> This loads logo.png from the images folder.
  • Parent folder: If the image is one folder above your HTML file: <img src="../logo.png" alt="Logo"> .. means “go up one folder”.

Absolute Paths (external resources)

Absolute paths give the full web address (URL). Use them only for external websites/resources.

Example:

<a href="https://google.com">Google</a>
<img src="https://upload.wikimedia.org/.../html5.svg" alt="HTML5">

Rule: Use relative paths for your own project, absolute paths for external links.

2️. HTML Boilerplate

The HTML boilerplate is the basic structure every webpage starts with. It ensures the page runs properly in all browsers. 

<!DOCTYPE html>  ← Tells the browser this is HTML5
<html lang="en"> ← Root element, language is English
<head>           ← Metadata (info about the page, not shown directly)
  <meta charset="UTF-8"> ← Supports all characters (English, Hindi, Emoji )
  <meta name="viewport" content="width=device-width, initial-scale=1.0"> ← Makes page mobile-friendly
  <title>My First Page</title> ← Title shown in browser tab
</head>

<body>
  Content visible to users goes here
</body>
</html>

✔ Prevents quirks mode (old browser errors)
✔ Ensures proper text encoding (UTF-8)
✔ Mobile responsive by default
✔ SEO & accessibility ready

3️. Multipage Websites

Websites usually have multiple pages like Home, About, Contact. Each page is a separate HTML file, linked with <a> tags.

Example folder structure:

project/
 ├── index.html   ← Home Page
 ├── about.html   ← About Page
 └── contact.html ← Contact Page

Navigation links:

<a href="index.html">Home</a>
<a href="about.html">About</a>
<a href="contact.html">Contact</a>

Tip: Keep header and footer same across all pages for consistency.

4️. Important Tags

These are frequently used structural tags in HTML:

  • <div> → Groups elements into blocks. (Block-level container)
  • <span> → Highlights text inline. (Inline container)
  • <header>, <main>, <footer> → Define page structure (semantic HTML).
  • id → Unique identifier for one element (use once). class → Reusable label for multiple elements.

Example:

<header>
  <h1>My Website</h1>
</header>

<main>
  <div class="content">
    <p>This is my paragraph with a <span style="color:red">highlighted word</span>.</p>
  </div>
</main>

<footer>
  <p id="copyright">© 2025 My Website</p>
</footer>
