Day 5: Forms in HTML

Collecting and Submitting User Information

Lesson Video

1️. Why Forms?

Forms make the web interactive. Without them, no login, search, comments, or shopping carts would work. A form:

  • Displays fields (text, checkboxes, dropdowns)
  • Packages data neatly
  • Sends it to a server

2️. Input & Label

<form>
  <label for="firstName">First Name:</label>
  <input type="text" id="firstName" name="firstName">
</form>

id links input with label. name is the key sent to the server.

3️. Submit Button

<button type="submit">
  <strong>Submit</strong> Form
</button>

Better than <input type="submit"> because you can style and add HTML inside.

4️. Radio Buttons (Select One)

<label>T-Shirt Size:</label>
<input type="radio" id="s" name="shirtSize" value="S">
<label for="s">Small</label>
<input type="radio" id="m" name="shirtSize" value="M">
<label for="m">Medium</label>

Same name = one group. Only one can be selected.

5️. Checkboxes (Select Many)

<label>Pizza Toppings:</label>
<input type="checkbox" id="pep" name="toppings" value="pepperoni">
<label for="pep">Pepperoni</label>
<input type="checkbox" id="mush" name="toppings" value="mushrooms">
<label for="mush">Mushrooms</label>

Multiple can be chosen. All share the same name.

6️. Textarea

<label for="comments">Special Instructions:</label>
<textarea id="comments" name="comments" rows="4" cols="40"></textarea>

Good for long text like addresses or comments.

7️. Dropdown

<label for="country">Country:</label>
<select id="country" name="country">
  <option value="">--Choose--</option>
  <option value="in">India</option>
  <option value="us">USA</option>
</select>

Compact way to choose from many options.

8️. Special Inputs (HTML5)

Password: <input type="password" name="pass">
Age: <input type="number" min="18" max="99">
DOB: <input type="date" name="dob">
Color: <input type="color" name="favColor">
File: <input type="file" name="upload">

Smarter fields with built-in validation.
