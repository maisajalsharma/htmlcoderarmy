HTML MEDIA - DAY 6

Lecture Video

1. The Web is More Than Text and Images

Webpages can deliver any content, not just text or images.
Earlier, videos and audio relied on third-party plugins (Flash, QuickTime), which were insecure and inconsistent.

2. HTML5 Solution: <video> and <audio>

Native, plugin-free tags for embedding media.
Standardized across modern browsers and devices.

3. Basic Video

<video> tag displays a video.
Adding controls enables play/pause, volume, timeline, and fullscreen.
width & height control the player size.

4. Video Attributes

  • autoplay: Plays video automatically (usually requires muted).
  • loop: Restarts video when finished.
  • poster: Displays a thumbnail before playback.
  • muted: Disables audio, useful for background videos.

5. Audio

<audio> works like <video> but without visuals.
Attributes: controls, autoplay, loop, muted.

6. Browser Compatibility

Not all browsers support the same formats.
Common formats: .mp4 (H.264), .webm (VP8/VP9), .ogg (Theora).
Use <source> inside <video> or <audio> for multiple formats.
Fallback text can be displayed for unsupported browsers.

7. Accessibility

<track> adds captions/subtitles:
captions for deaf users
subtitles for translation
descriptions for visually impaired
srclang specifies language, label names the track.

8. Methods to Embed Video

Robust <source> Method: Supports multiple formats for maximum compatibility. Recommended for public websites.
Simple src Method: Single video file, simple but less compatible. Suitable for internal projects.

9. HTML Symbols

Use entity names (&copy;) or entity numbers (©) to display special symbols.
Common symbols: © &copy;, ® &reg;, ™ &trade;, < &lt;, > &gt;, & &amp;, " &quot;, € &euro;
Entities prevent the browser from misinterpreting characters as HTML tags.
