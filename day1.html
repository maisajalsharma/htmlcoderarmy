1. Core Principles (Problem ‚Üí Solution)

  1. Need to share info:Computers exchange data with signals (1s & 0s).
  2. Problem of scale: Can‚Äôt connect every machine directly ‚Üí use a Switch (LAN).
  3. Connecting networks: LANs connect with Routers ‚Üí many routers = Internet.
  4. Addressing: Every device has a unique IP Address.
  5. Reliability: Data split into Packets, checked by TCP/IP.

2. IP Addresses

IPv4

  • 32-bit address, e.g. 172.217.16.142
  • ‚âà 4.3 billion addresses ‚Üí shortage caused public/private split

Public vs Private

  • Public: Globally unique, assigned by ISP
  • Private: For LAN use, e.g. 192.168.x.x
  • Common ranges: 10.x.x.x, 172.16‚Äì31.x.x, 192.168.x.x

IPv6

  • 128-bit, written in hex groups
  • Example: 2001:0db8:85a3::8a2e:0370:7334

3. Packets & Protocols

Packet = data envelope containing:

  • Payload (actual data)
  • Source & Destination IP
  • Sequence number

TCP/IP: Ensures reliable & ordered delivery with checksums + retries.

4. MAC Address

Unique hardware ID (48-bit) burned into devices.

Formats:

  • 3C:22:FB:A3:B4:C5 (colon)
  • 3C-22-FB-A3-B4-C5 (hyphen)
  • 3c22.fba3.b4c5 (period)

Find it:

  • Windows: ipconfig /all or getmac /v
  • macOS/Linux: ifconfig

5. Port Numbers

Range: 0‚Äì65535

  1. 0‚Äì1023: Well-known (HTTP 80, HTTPS 443, SSH 22)
  2. 1024‚Äì49151: Registered (apps, dev ports like 3000)
  3. 49152‚Äì65535: Dynamic/Ephemeral (temporary client ports)

Common: 80 (HTTP), 443 (HTTPS), 22 (SSH), 3000 (Node/React), 27017 (MongoDB)

6. DNS Flow

  1. Check cache (browser ‚Üí OS ‚Üí router)
  2. Ask resolver (ISP / public like 8.8.8.8)
  3. Resolver queries hierarchy:
    • Root ‚Üí TLD (.com, .org, etc.)
    • TLD ‚Üí authoritative server
    • Authoritative ‚Üí final IP
  4. Resolver caches result
  5. Browser connects to IP + port (e.g. 443 for HTTPS)

7. TL;DR (Quick Recap)

  • Switch ‚Üí connects devices in LAN
  • Router ‚Üí connects networks
  • IP ‚Üí digital address
  • MAC ‚Üí hardware ID
  • Packets + TCP/IP ‚Üí reliable data transfer
  • DNS ‚Üí domain name ‚Üí IP
